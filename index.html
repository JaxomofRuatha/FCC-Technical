Introduction

SAS is a powerful data analysis suite that takes practically any data source and processes it through coded steps into a form where anaytics and reporting can come to insightful conclusions.

Program Steps

Programs in SAS contain two types of steps:

  1. DATA step - allows for reading or writing data, manipulating the data, and performing calculations.
  2. PROC step - prompts SAS to run procedures to perform certain functions, including producing reports, summaries, and visualizations of data.
There are also global statements in SAS, which persist throughout the current environment and between steps. Every statement must end in a semicolon to be valid.

Data Types and Properties

There are only two data types in SAS: character and numeric. These are stored in 8 bytes unless specified by the coder otherwise.

Basic Syntax

A DATA step will start with the word data and end with a statement of run;. A PROC step will start with the word proc and also end with run;. Good practice is to proceed any step that produces output with the word title to create a global statement labeling the data for readability. Within these steps, some of the additional statement keywords include:

  • INFILE - this statement will direct SAS to where data values are coming from
  • INPUT - will process the raw data from the infile as determined by the format given (more in the Input and Formatting section).
  • DATALINES - signals SAS to begin reading the data on the next line.
  • LENGTH - used before INPUT statement to dictate maximum length of variable name. Ex: LENGTH [variable] $12 creates variable with maximum length of 12.
  • SET - statement used to copy a data set.
  • IF - statement used to filter data, only keeping observations meeting the conditions specified.
  • LIBNAME - defines the creation of a new library (takes definitions for name and directory)
  • LABEL - defines a data set label, which is displayed in certain outputs.
  • COMMENT - keyword can be used to denote a strictly comment statement. This can also be done using *; and /* */

Windowing Environment

SAS is divided into several different windows, each providing a specific portion of the current environment. These are divided up as follows:

  • Code - This is where you will write the code to be executed by SAS. It has a "Submit" button to allow you to run the code.
  • Output - This window will show the results of running the parts of your code intended to generate output.
  • Log - This window displays the dataflow and any error or warning messages resulting from the running of your program, as well as any technical details like processing time.
  • Explorer - This section allows for exploring of libraries and system directories containing data sets and file trees.

Input and Formatting

By default, the infile will be read and parsed based on blanks as a delimeter of data inputs. This can be changed to several different options depending on the type of infile being used and original formatting of the data to be read.

Libraries

Libraries show up under the "Libraries" folder within the Explorer window. You can reference a data set within a library using the notation [LIBRARYNAME].[DATASETNAME]

There are two different types of libraries in SAS: WORK and PERMANENT. The WORK library is a temporary library with all the working data sets, and will be deleted after the program is closed. The remaining libraries are part of the PERMANENT library, and will be saved after closing.

Procedure Statements

PROC Export - allows to export SAS data file into an external file like an Excel spreadsheet or text file. This statement takes 4 arguments:

  1. DATA - selects the data set to use for exporting.
  2. OUTFILE - determines the file path that will be created when exporting.
  3. REPLACE - option to overwrite the exporting file if it already exists on the stated path (recommended to always use).
  4. DBMS - specifies the file type for the export (Ex: xls for Excel spreadsheet).

PROC Print - prints variables for observations in a data set, with no options will print all of each, but has options to filter what prints in the report.

PROC Contents - Creates a description of the data set instead of the data values.

PROC Datasets - Generates a detailed listing of the members of a data library.

PROC Options - Lists the current SAS system settings, takes arguments to make changes.

PROC Means - Provides descriptive statistics for numeric variables in a data set.

PROC Transpose - Restructures values for output, turning selected variables into observations.

PROC SQL - Allows for the usage of SQL query language in obtaining filtered portions of a data set or modifying/creating data sets.
