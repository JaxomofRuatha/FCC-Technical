Introduction

SAS is a powerful data analysis suite that takes practically any data source and processes it through coded steps into a form where anaytics and reporting can come to insightful conclusions.

Program Steps

Programs in SAS contain two types of steps:

  1. DATA step - allows for reading or writing data, manipulating the data, and performing calculations.
  2. PROC step - prompts SAS to run procedures to perform certain functions, including producing reports, summaries, and visualizations of data.
There are also global statements in SAS, which persist throughout the current environment and between steps. Every statement must end in a semicolon to be valid.

Data Types and Properties

There are only two data types in SAS: character and numeric. These are stored in 8 bytes unless specified by the coder otherwise.

Basic Syntax

A DATA step will start with the word data and end with a statement of run;. A PROC step will start with the word proc and also end with run;. Good practice is to proceed any step that produces output with the word title to create a global statement labeling the data for readability. Within these steps, some of the additional statement keywords include:

  • infile - this statement will direct SAS to where data values are coming from
  • input - will process the raw data from the infile as determined by the format given (more in the Input and Formatting section).

Windowing Environment

SAS is divided into several different windows, each providing a specific portion of the current environment. These are divided up as follows:

  • Code - This is where you will write the code to be executed by SAS. It has a "Submit" button to allow you to run the code.
  • Output - This window will show the results of running the parts of your code intended to generate output.
  • Log - This window displays the dataflow and any error or warning messages resulting from the running of your program, as well as any technical details like processing time.
